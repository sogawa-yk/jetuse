# タスク: HBD-03 合成エンジン（sample-app×AI部品×connector→デモ構成）＋プレビュー

## ゴール
確定した推薦構成から、実行可能なデモ構成（sample-app 定義＋AI組込スロットのバインド＋コネクタ＋UI/シード）を
**合成**し、デプロイ前にプレビューできるようにする。

## 対象 area
both（api：合成エンジン＋プレビュー定義生成 / web：プレビュー表示）

## 依存
HBD-01（推薦構成）。SBA-01/02 の sample-app レジストリ・AI組込FW（`sample_app_registry` / `ai_runtime`）を素材として再利用。

## 仕様参照
docs/enhance/202607-demo-platform-plan.md §3（全体像）/ §5.1 / §10「HBD-03」、
docs/enhance/202607-hearing-flow.md §4（推薦3要素）。

## 受け入れ条件（検証可能な述語で書く）
- [ ] 合成エンジン: 推薦（主SBA＋AI部品セット＋コネクタ＋UI＋シード方針）を入力に、`sample_app` 定義へ AI スロットを実行時バインドした**デモ構成オブジェクト**を生成する
- [ ] AI部品は既存の capability レジストリ（`ai_runtime`）から束縛し、未束縛 capability は構成に含めない（または明示エラー）
- [ ] シード方針（サンプルシード / GenAI 業界寄せ / 後差替）を構成に反映する
- [ ] プレビュー: 合成したデモ構成（画面・組込点・使うAI・データ）を実行せずに描画でき、配布表現（再検証可能）を壊さない
- [ ] api lint（ruff）/ web build・vitest・eslint クリーン。合成の単体テスト（代表構成＋未束縛/不整合ケース）を追加

## E2E シナリオ（実環境 / jetuse-dev・複数）
完了ゲートで loop 環境へデプロイ（DB は専用スキーマ `JETUSE_HBD-03` で隔離）し、証跡を `runs/<run-id>/e2e/` に残す。
- [ ] シナリオ1: 推薦（例 SBA-A＋{RAG-QA,要約,分類}＋Slack＋チャットUI＋FAQシード）→合成→プレビューが構成通りに描画される
- [ ] シナリオ2: NL2SQL を含む推薦（SBA-B）で、AIスロットが実 capability にバインドされ、プレビューに組込点が現れる
- [ ] シナリオ3（境界）: 未束縛 capability や許可外組合せを含む推薦で合成が安全に失敗/警告する（HBD-04 の前段チェックに渡せる形）
- [ ] 実施不能な範囲は `runs/<run-id>/e2e/SKIPPED.md` に理由明記

## 成果物
`jetuse_core` の合成エンジン＋プレビュー定義 / web プレビュー画面 / `docs/verification/HBD-03.md`

## 非ゴール / 制約
- 厳密な合成バリデーション（許可組合せ・必要ケイパ・権限スコープの網羅判定）は HBD-04。本タスクは「構成生成＋描画」と前段の整合チェックまで。
- 実デプロイ（コンテナ配備）は S4。本タスクのプレビューは既存 loop 基盤上の宣言定義レンダリング。
- 既存リソースは参照のみ。spec-driven。コミット/PR/push は人間承認後。
