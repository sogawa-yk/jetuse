# タスク: SP1-02 demos 最小レジストリ + DemoContext seam

## 目的
「呼び出しが必ずデモを名乗り、そのデモの箱の中しか触れない」を保証する継ぎ目（seam）を敷く
（specs/17 §5）。所有権検証は信頼境界 — **fail-closed** で実装する。

## 仕様参照
specs/17-demo-platform-redesign.md §5（(user, demo) スコープの継ぎ目）・§9（受け入れ条件）

## 対象 area
api

## 前提（依存タスク / 人間の事前作業）
なし（第1波・SP1-01 と並列可）。

## 作業内容
- migration `packages/api/jetuse_core/migrations/016_demos.sql` を新設。**所有権検証に必要な最小列のみ**:
  `demos(id VARCHAR2, owner_sub VARCHAR2, name VARCHAR2, visibility VARCHAR2 DEFAULT 'private',
  created_at TIMESTAMP)`。usecases テーブルの流儀（owner_sub / visibility）を踏襲する。
  ※ config・箱のプロビジョニング状態などの列は **SP2 で追加**する（specs/17 §8 スコープ境界）。
- `packages/api/jetuse_core/demos.py` を新設（usecases.py の流儀を踏襲した最小リポジトリ）:
  - `create_demo(owner: str, name: str) -> dict`（id は uuid4。E2E とテストのシード用）
  - `get_demo(demo_id: str) -> dict | None`（owner_sub / visibility を含む生取得。呼び出し側で認可判定）
  - `delete_demo(owner: str, demo_id: str) -> bool`（所有者のみ。SQL で強制）
- `packages/api/service/demo_context.py` を新設:
  - `@dataclass DemoContext: demo_id: str / owner_sub: str / namespace: str`。
    `namespace = f"demo_{demo_id}"` — RAG・会話の名前空間キー、将来の DB スキーマ名の元
    （specs/17 §5「箱の分け方」）。
  - FastAPI 依存関数 `require_demo(demo_id: str, user: AuthContext = Depends(require_user)) -> DemoContext`:
    demo が存在しない → **404**。存在するが `owner_sub != user.subject` かつ `visibility != 'public'`
    → **404**（存在秘匿のため 403 でなく 404 に統一）。それ以外は DemoContext を返す。
- 単体テスト `packages/api/tests/test_demo_context.py`:
  - 所有者は DemoContext を得る（namespace = `demo_<id>`）。
  - 他ユーザーは 404。存在しない id は 404。`visibility='public'` は他ユーザーも通る。
  - 認証なし（AUTH_REQUIRED=true 相当）は 401。
