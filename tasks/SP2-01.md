# タスク: SP2-01 Demo エンティティ本格化 + CRUD ルート

> **暫定チケット**: 受け入れ条件の確定は SP2-00（specs/18 承認）後。以下は specs/17 §6 からの骨子。

## 目的
SP1-02 の最小 demos テーブル（所有権検証のみ）を一級エンティティに拡張し、Demo の CRUD を API として提供する。
SP3（ビルダー）が「生成したデモを保存する」着地先になる。

## 仕様参照
specs/18-sp2-demo-tenancy.md（SP2-00 成果物）、specs/17 §6

## 対象 area
api

## 前提（依存タスク / 人間の事前作業）
SP2-00（specs/18 承認）。

## 作業内容（骨子 — specs/18 で確定）
- migration `017_demos_v2.sql`: demos に `config`（JSON/CLOB）・`status`（provisioning/ready/failed）・
  `updated_at` を追加（列の最終形は specs/18）。
- `jetuse_core/demos.py` 拡張: `list_demos(owner)` / `update_demo(owner, id, ...)` / `set_status(...)`。
  usecases の流儀を踏襲、SQL で所有者強制。
- routes `service/routes/demos.py` に CRUD を追加: `GET /api/demos`（自分の一覧）、`POST /api/demos`、
  `GET /api/demos/{demo_id}`（所有者 or public）、`PATCH /api/demos/{demo_id}`・`DELETE /api/demos/{demo_id}`
  （所有者のみ）。存在秘匿 404（SP1 の require_demo / require_demo_owner を流用）。
- 単体テスト: CRUD 往復・越境 404・public の読み取り可/書き込み不可・401。
