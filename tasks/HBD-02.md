# タスク: HBD-02 ダイアログ式ヒアリングUI（順次Q&A・回答保存・進捗）

## ゴール
フィールドSAが、ヒアリングメモ貼付→順次Q&A（デフォルト提案つき）→回答確定までを画面で行える
ダイアログ式UIを用意し、各回答を HBD-01 のAPIへ構造化保存する。

## 対象 area
web（＋ HBD-01 の回答保存/recommend API への疎通）

## 依存
HBD-01（質問スキーマ＋セッション/回答API＋推薦API）。

## 仕様参照
docs/enhance/202607-demo-platform-plan.md §5.1（スタンダードフロー）、
docs/enhance/202607-hearing-flow.md §2（全体フロー）/ §3（質問文言・選択肢）。

## 受け入れ条件（検証可能な述語で書く）
- [ ] ルート（例 `/builder/standard` または `/hearing`）を追加し、左ナビ/ホームから到達できる
- [ ] 入力ステップ: ヒアリングメモを貼ると HBD-01 のGenAI補助で Q1..Q6 のデフォルト選択が**提案**として埋まる（SAは確認・修正のみ）
- [ ] Q1..Q6 を順次表示・分岐（hearing-flow §3 の分岐例: Q2「業務DB」×Q3「集計」→SBA-B格上げ 等）し、各回答を保存。進捗（n/6）を表示
- [ ] 「確定」で推薦構成（主SBA＋AI部品＋コネクタ）を画面に提示する（HBD-03 の合成に渡す手前まで。提示＝ブラックボックス化しない）
- [ ] `npm run build` 成功・`vitest`・eslint がクリーン。主要分岐の UI テストを追加

## E2E シナリオ（実環境 / jetuse-dev・複数）
完了ゲートで loop 環境へデプロイ（`ops/dev-env-up.sh loop`）し、証跡（スクショ/HTTP）を `runs/<run-id>/e2e/` に残す。
- [ ] シナリオ1: メモ貼付→デフォルト提案表示→Q1..Q6 を回答→確定→推薦構成が表示される（実APIと往復、回答が ADB に保存される）
- [ ] シナリオ2（分岐/境界）: Q2「業務DB」×Q3「集計・分析」で SBA-B が主役に格上げされる／途中離脱→再開でセッション回答が復元される
- [ ] 実施不能な範囲は `runs/<run-id>/e2e/SKIPPED.md` に理由明記
