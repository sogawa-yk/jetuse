# タスク: SP3-05 ビルダー UI（ヒアリング→プレビュー→保存）+ デモ産出 E2E

> 受け入れ条件は specs/19 準拠で確定（SP3-00 起草。**specs/19 の人間承認が着手条件**）。以下は暫定。

## 目的
フィールドSA 向けのビルダー UI を作り、ヒアリング→デモ設計→生成→**プレビュー→保存**の一連を
1画面体験にまとめる。パイプライン（SP3-01〜04）を束ねて `Demo` として産出する縦切りを完成させる。

## 仕様参照
specs/19-sp3-builder.md（ビルダー UX・産出フロー）、specs/17 §1・§6、旧 UC-03（非開発者が5分で作成）。

## 対象 area
web（+ 束ねる薄い api glue があれば api）

## 前提（依存タスク / 人間の事前作業）
SP3-03（フロント生成・配信）＋ SP3-04（データ投入）。

## 作業内容（specs/19 で確定・暫定）
- ビルダー UI: ①ヒアリング（NL 対話）→ ②生成されたデモプランの確認/微調整 → ③生成の進行表示 →
  ④**プレビュー**（生成デモを `/api/demos/{id}/...` で開く）→ ⑤**保存**（`Demo` として確定）。
- Internal 面（AUTH 前提）。既存 SPA/デザイントークンに準拠（UI-02 等の流儀）。
- `npm run build` 成功まで（フロント完了条件）。UI ロジックの単体テスト（状態遷移・エラー表示）。

## 完了条件（検証可能な述語で）
- [ ] ビルダー UI からヒアリング→生成→プレビュー→保存が通り、保存後に一覧（SP2 CRUD）へ現れる。
- [ ] `npm run build` 緑・UI 単体テスト緑・既存 web 回帰なし。
- [ ] **デモ産出 E2E（デプロイ済み環境・ステージの総合シナリオ）**: 「フィールドSA がヒアリングに答える →
  デモ設計 → 生成 → データ投入 → 保存 → 生成デモが開き JetUse API をデモスコープで叩ける」を
  ブラウザで実機確認し `runs/<run-id>/e2e/` に記録（旧 UC-03「5分で作成」を Internal ビルダーで充足）。

## E2E シナリオ
- デプロイ済み環境（gateway）に対し、ビルダー UI から新規デモを1件作り切り、生成デモの動作まで確認。

## 成果物
- コード（ビルダー UI・glue・テスト）・`docs/verification/SP3-05.md`（ステージ総合 E2E 証跡込み）。

## 非ゴール / 禁止事項
- マーケット公開/配布（SP4）。バックエンド生成。認証情報のコミット禁止。
