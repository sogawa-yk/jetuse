# タスク: BE-03 コネクタ実行の実体化（Slack コア / CON-02・03 実E2E）

## ゴール
コネクタの実行経路を fail-closed のモックから**実体**へ。`connector_runtime` の `http_caller` を実 HTTP に、
secret を **Vault から解決**し、`/api/platform/connector/invoke` の **501 を解除**して実 invoke を通す。
コア Slack コネクタで実ワークスペースへの実投稿を成立させる。

## 対象 area
api（jetuse_core/plugins: connector_runtime / slack_connector_builtin ／ service/routes/platform.py）

## 仕様の正
- specs/16-platform.md §12（connector / invoke）・§13（Platform broker）。ADR-0014（スコープ・テナント境界・秘密の扱い）。
- 親計画 §10 ステージ6 BE-03。[[stage3-real-slack-e2e-pending]]。
- 現状: `connector_runtime.py:139-147` `_denied_http_caller` が既定で拒否（実 SaaS 呼出なし・テストはモック注入）。
  `platform.py:211-214` の connector/invoke は **501**（実行は CON-02/03 へ繰延と明記）。

## 受け入れ条件（検証可能な述語）
- [ ] `invoke_connector_action` が実 `http_caller`（live httpx）で SaaS API を呼ぶ。secret は Vault 解決（実値はコード/DBに置かない）。
- [ ] `/api/platform/connector/invoke` が 501 でなく、スコープ認可・テナント境界・二重閉包を満たした上で実 invoke を返す。
- [ ] Slack コア: `chat.postMessage` 等が実トークンで成功（トークンは Vault・本体のみ保持）。fail-closed は不正時のみ。
- [ ] 既存の合成バリデーション（connector スコープ宣言・許可組合せ）を壊さない。
- [ ] `pytest`（connector 関連・モック経路は維持）全パス。lint/型クリーン。

## E2E シナリオ（実環境 / jetuse-dev・最低2本）
- [ ] 正常系（**実 Slack**）: 合成→ブローカートークン→connector/invoke で実ワークスペースへ投稿成功。証跡（API応答/スクショ）。
- [ ] 異常系: スコープ外/未承認/secret 欠如で fail-closed（投稿されない・適切なエラー）。
- [ ] 実 Slack/Vault を要するため未実施範囲は `SKIPPED.md` に明記し報告で提示。

## 非ゴール / 制約
- Slack 以外のコネクタ網羅（コアの実体化に集中）。
- **人間ゲート（濃い）: Vault/IAM・実 Slack ボットトークン・実ワークスペース**。自走は設計＋mock E2E まで、実接続は報告で一括提示。
- 秘密を L3 デモへ配らない（ブローカー一本／D5）。spec 外は ADR 案。
