# タスク: BE-08 認証付きMCPサーバー登録

## ゴール
`POST /api/agent/mcp-servers` の**認証付き**登録（現状 501）を実体化する。MCP の認証資格情報を **Vault に保持**し、
チャット/エージェントから認証付き MCP を実利用できるようにする。

## 対象 area
api（service/routes/agents.py ／ jetuse_core: mcp_servers / Vault 連携）

## 仕様の正
- specs/16-platform.md（MCP / connector）。ADR-0014（秘密の扱い・本体のみ保持）。
- 親計画 §10 ステージ6 BE-08。
- 現状: `agents.py:104-114` 認証付き MCP 登録は **501**（「Vault 書込権限の追加が必要」）。無認証 MCP は実動（INSERT＋SSRF ガード）。

## 受け入れ条件（検証可能な述語）
- [ ] 認証付き MCP 登録が 501 でなく、資格情報を **Vault に書込**（実値は DB/コードに置かない）。SSRF/URL ガードは維持。
- [ ] 登録した認証付き MCP がチャット/エージェントのツール実行で実利用できる。
- [ ] 無認証 MCP 経路（既存）を壊さない。
- [ ] `pytest`（mcp_servers / agents route）全パス。lint/型クリーン。

## E2E シナリオ（実環境 / jetuse-dev・最低2本）
- [ ] 正常系: 認証付き MCP を登録→Vault に secret 保存→チャットから実呼出成功。証跡。
- [ ] 異常系: 不正 URL/権限欠如で fail-closed。
- [ ] Vault 書込 IAM を要するため未実施は `SKIPPED.md` に明記し報告で提示。
