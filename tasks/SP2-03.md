# タスク: SP2-03 DemoContext 解決先の実装 + dbchat デモスコープ化

> **暫定チケット**: 受け入れ条件の確定は SP2-00（specs/18 承認）後。以下は specs/17 §5-6 からの骨子。

## 目的
`DemoContext` の解決先（namespace → 箱の実体: DB スキーマ名・RAG ストア・会話紐付け）を実装し、
SP1-03 で非ゴールだった **dbchat のデモスコープ化**（`/api/demos/{demo_id}/dbchat/...`）を SP1-03 と同型で敷く。
これで 8 能力のうちデモの箱を持つ主要 3 系統（chat / rag / dbchat）が縦に揃う。

## 仕様参照
specs/18-sp2-demo-tenancy.md（SP2-00 成果物）、specs/17 §5・§6、SP1-03（同型の先例）

## 対象 area
api

## 前提（依存タスク / 人間の事前作業）
SP2-02（`demo_<id>` スキーマが実在すること）。

## 作業内容（骨子 — specs/18 で確定）
- `DemoContext` に箱の実体解決を追加（例: `db_schema` プロパティ = `demo_<id>`。形は specs/18）。
- `routes/demos.py` に dbchat 縦切りを追加: NL2SQL / execute / schema をデモスキーマに対して実行
  （既存ハンドラを namespace/スキーマ引数の共有内部関数へ整理 — SP1-03 の rag と同じやり方）。
- capabilities の `dbchat` ディスクリプタにデモスコープルートを追記（routes 実在テストが検証）。
- 単体テスト: 所有者の NL2SQL 往復（モック）・越境 404・user 単位 dbchat の回帰なし・箱の分離。

## 完了条件（暫定）
- [ ] テスト全緑・回帰なし・ruff クリーン。SP1-01 の routes 実在テストが更新後も緑。
- [ ] 実環境 E2E: 実 ADB のデモスキーマに投入したデータへ `/api/demos/{id}/dbchat` で日本語質問→
  正しい SELECT と結果が返る／別ユーザー 404／既存 `/api/dbchat` 回帰なし。`runs/<run-id>/e2e/` に記録。
