# タスク: SP3-04 サンプルデータ生成 + 箱への投入（demo_<id>）

> 受け入れ条件は specs/19 準拠で確定（SP3-00 起草。**specs/19 の人間承認が着手条件**）。以下は暫定。

## 目的
デモプラン（SP3-02）のデータスキーマから、サンプルデータを生成してデモの箱 `demo_<id>` スキーマ
（SP2-02 のプロビジョニング機構）へ表作成＋投入する。dbchat / rag がそのデータを引ける状態にする。

## 仕様参照
specs/19-sp3-builder.md（データ生成/投入）、specs/18（`demo_<id>` プロビジョニングと後始末）、
specs/17 §5（箱の分け方）・§6。

## 対象 area
api

## 前提（依存タスク / 人間の事前作業）
SP3-02（デモプランのデータスキーマ）。SP2-02 の `demo_<id>` プロビジョニング/後始末が土台。

## 作業内容（specs/19 で確定・暫定）
- デモプランのデータスキーマから、`demo_<id>` スキーマに表を作成し、LLM/生成ロジックで
  もっともらしいサンプルデータを生成して投入する。件数・型はプラン準拠。
- dbchat（NL→SQL）がそのスキーマを対象に引けるよう、SP2-03 の DemoContext 解決（スキーマ名の束ね）と
  整合させる。RAG 用の文書サンプルが要るなら該当デモのベクタストアへ投入。
- **後始末整合**: デモ削除（SP2-02 の後始末）で本タスクが作った表/データも消えること（孤児を作らない）。
- テスト: プラン→表作成/投入の冪等性・`demo_<id>` 隔離（他デモ/共有スキーマを触らない）・
  削除後始末で消えること・投入データに対する dbchat 往復。

## 完了条件（検証可能な述語で）
- [ ] デモプランから `demo_<id>` に表＋サンプルデータが投入され、dbchat がそのデータを引ける。
- [ ] デモ削除で本タスク由来の表/データが後始末される（孤児なし）。単体テスト全緑・ruff クリーン・回帰なし。
- [ ] 実環境 E2E（デプロイ済み環境の実 ADB・`JETUSE_SP3_04` or `demo_<id>` 隔離）:
  プラン→投入→dbchat 照会→デモ削除で消えるまでを `runs/<run-id>/e2e/` に記録。

## E2E シナリオ
- デプロイ済み環境で、プラン1件→`demo_<id>` 表作成/投入→dbchat で自然言語照会→デモ削除で後始末完走。
