# specs/15: Phase 11 エンタープライズ硬化（SEC-01〜03 / OPS-01〜02）

## [SEC-01] SAMLフェデレーション手順書
- 成果物: `docs/setup/saml-federation.md`（Entra ID / Google Workspace の2系統）
- エージェントが起草、**人間がIdP側設定と実機検証**（Identity Domain側もコンソール作業=人間）
- 範囲: SAML IdP登録・メタデータ交換・属性マッピング・JITプロビジョニング・IdPポリシー・切り戻し手順
- 完了条件: 手順書レビュー（実機検証はIdPテナント準備でき次第）

## [SEC-02] ガードレールと監査ログ
- 入出力モデレーション: OCIで利用可能な方式を実機調査して採用を決める
  （候補: 互換APIのmoderationエンドポイント有無 / LLM自己判定ガード / プロンプト指示）
- 監査ログ: **誰が・どの機能・どのモデル・トークン数** をADBに記録
  - 既存usage_log（会話単位）を拡張し、ユースケース実行・RAG・NL2SQL・議事録・エージェントも対象に
  - 管理向け集計ビュー（OPS-01の材料）
- 完了条件: モデレーション方式の決定（比較表）+ 監査ログがE2Eで記録される

## [SEC-03] IP制限・レート制限
- API GWのusage plan / rate limiting ポリシーで実装（WAFは比較のみ）
- 完了条件: GW経由でレート制限の429を実測

## [OPS-01] 管理ダッシュボード
- 利用状況（ユーザー/機能/モデル別トークン・概算コスト）をSPAの管理ページに表示
- データ源はSEC-02の監査ログ
- 完了条件: 管理ページで当月利用が見える

## [OPS-02] 可観測性
- OCI Logging（CI/Functions/GWのログ集約）+ アプリログのJSON Lines整備（実装済み）の接続
- 完了条件: Loggingでアプリエラーが検索できる
