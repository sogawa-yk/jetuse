## [CI-01] GitHub Actions

### 目的

mainおよびPRに対して lint / test / build を自動実行し、品質ゲートをローカル運用からCIへ移す。

### 段階導入

1. **第1段（本タスク）**: 検証ワークフロー
   - api: ruff check + pytest（Python 3.12）
   - web: npm ci + lint + build（Node 22）
   - terraform: fmt -check + validate（providerダウンロードのみ、認証不要）
2. **第2段（Phase 2以降・別タスク）**: コンテナビルド → OCIR push → Container Instances更新（dev自動）。OCI認証はGitHub Secrets（APIキー or OIDC連携）の設計を人間と合意してから

### 完了条件

- [ ] PR/maINプッシュでワークフローが走り、全ジョブgreen（GitHub上で実確認）

### 運用ルール（CLAUDE.md更新事項）

- リモート接続済みのため、以後のタスクは**ブランチpush → PR → マージ**。PRには動作確認ログを添付
- gh CLI導入と認証が整うまでは、エージェントはブランチpushまで行い、PR作成リンクを提示する
