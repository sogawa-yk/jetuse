- Responses直結が不可の理由: SDKは `{"role","content"}` の簡易input形式を送るが、OCIは
  `type:"message"` + 型付きcontentの厳格スキーマを要求（`Invalid 'input'`）。
  さらに `OpenAi-Project` ヘッダが無いと**「Compartment ID must be provided」という誤誘導エラー**になる
  （切り分けに時間を要した。既知quirkの再確認 — ヘッダはあるのに無いと言われたら別原因を疑うこと）
- **新発見: gpt-oss-120bはchat completionsエンドポイントでも動く**（従来Responses系として登録。
  function callingも可だが、引数JSONに空キー等の崩れが出ることがある → 寛容化で対処）
- tracingは `set_tracing_disabled(True)` 必須（OpenAI本家へ送信しようとする）

   文書ありストアで5件ヒット・score付き）。文書なしユーザーには案内文を返す動作も確認

UIは既存の承認カードを両プロトコル対応に拡張（SDK系は可否のみ返送）。残るnative限定はcode_interpreterのみ。
