# SP3-07 検証レポート: GitHub Actions による jetuse:dev 自動デプロイ

日付: 2026-07-09 / 実施: Claude（loop maker）/ 採点: Codex
関連: tasks/SP3-07.md、docs/guides/dev-environments.md「開発ブランチ push → jetuse:dev 自動デプロイ」

## 1. 何を作ったか

dev 派生の開発用ブランチ（`feat/**`・`fix/**`・`chore/**`）への push で、jetuse:dev の共有
プレビュー（RM スタック `jetuse-dev-app`）へ自動デプロイするパイプライン。

- **`.github/workflows/deploy-dev.yml`**: 各 step = `ops/deploy-dev-app.sh` のサブコマンド
  （ローカル検証とワークフローが同一手順）。`concurrency: dev-app-deploy`（cancel-in-progress）
  + スクリプト側の先行 RM job 完了待ちで共有スタックの apply を直列化。
- **`ops/deploy-dev-app.sh`**: image（build→OCIR push、tag=`<branch>-<short-sha>`）→
  update-stack（tf 構成 zip + `image_url` 変数。**既存変数はマージで温存**）→ apply
  （`oci resource-manager job` の plan→apply。ローカル terraform apply は権限層で不可 — SP2 実績）→
  spa（build→バケット CLI 同期）→ smoke。`seed-gen-shared` で ORASEJAPAN 材料を RM 変数へ
  一度だけシード（GitHub Secrets に共有テナンシ鍵を置かない）。
- **SP3-06 env の配備像**（作業内容2の確定）:
  - sign_proxy = **API プロセス内 mount**（`/gen-proxy`）。gateway は `/api/*` と SPA しか
    ルートしない → 非公開。VCN 内（SP3-08 生成 CI）からは `:8000/gen-proxy/v1`。
  - 自テナンシ署名は `AUTH_MODE=resource_principal` で RP（genai と同一流儀）。
  - ORASEJAPAN 材料 = RM sensitive 変数 → コンテナ env（`GEN_SHARED_*`）→ entrypoint.sh が
    `~/.oci/config` を冪等生成（未設定は no-op = fail-closed 403）。
  - dev-app tf に `generation_proxy_url` / `gen_shared_*` を宣言。**DB マイグレーションは
    自動デプロイに含めない**（共有 loop ADB のスキーマ変更は人間ゲート — review-1 B002 で
    当初の `RUN_DB_BOOTSTRAP=true` を撤去。適用手順は dev-environments.md）。

## 2. 実機で発見し修正した事実（Tips）

1. **旧 SP2 方式は SPA オブジェクトを terraform 管理**（dev-app/spa.tf の fileset(dist)）。
   CI パイプラインの zip に dist を含めないと fileset が空解決になり **apply が SPA を全削除**する
   （1回目デプロイで実発生 — SPA / 404）。→ spa.tf を廃止し、スタックは bucket+PAR のみ管理・
   オブジェクトは CLI 同期一本化・`apply 後に spa 同期`の順へ。RM zip の 11MB 上限と
   「SPA だけの更新に apply 不要」も同判断の理由。
2. **RM stack update の `--variables` は全置換** — 既存変数を `stack get` → jq マージで温存する
   （seed した ORASEJAPAN 材料が以降のデプロイで消えない仕組み）。
3. **ローカル terraform init の `.terraform.lock.hcl` を zip に入れない** — provider 版数を
   固定してしまう（zip から除外。RM は毎回 provider を解決 — 今回 v8.21.0）。
4. ビルダーのモデル一覧キーは**遅延チャンク** `assets/demobuilder-*.js` に入り、minify で
   **backtick 引用**になる。バンドル検査は index チャンクからチャンク名を追跡し引用符 3 種を許容。
5. deployed コンテナの entrypoint ログは
   `oci container-instances container retrieve-logs` で取得できる（プロファイル書き出し・
   migrations up to date・rag ledger reconcile を確認）。

## 6. 残課題（residual）

  自 IP 解決。
