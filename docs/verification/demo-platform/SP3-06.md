## 1. 何を作ったか

ビルダーのフロント生成（OpenCode バックエンド）LLM を UI から選択可能にした。
既定 gpt-oss-120b（自テナンシ大阪）+ 施主指定 7 モデル（ORASEJAPAN 共有テナンシ）。

- **署名プロキシ後継** `packages/api/jetuse_core/sign_proxy.py`（spikes/sp3_03_sign_proxy.py の後継。
  spike は歴史的成果物として残置・配備はしない）: 生成モデルルーティング表
  （モデル → {リージョンエンドポイント, auth プロファイル, compartment, api 種別}）。
  許可パスは `chat/completions` + `responses` の完全一致 2 つ（他 403）。SP3-03 の防御
  （本文 256KiB 上限・重複キー拒否 + 正規再シリアライズ・ヘッダ固定・SSE 素通し）は維持。
- **生成専用モデルレジストリ** `packages/api/jetuse_core/gen_models.py`: key → {oci_id, api,
  region, shared}（表示ラベルは web の i18n）。共用 MODELS には gpt-5 系を**足さない**（チャット UI へ漏れて自テナンシで
  404 になるため。test_gen_models.py が purity を検査）。プロキシ allowlist・runtime 検証・
  generate API 検証の単一真実源（旧 GENAI_MODEL_ALLOWLIST env は廃止）。
- **generate API**: `POST /api/builder/sessions/{sid}/generate` body に任意
  `{"model": <レジストリ key>}`。未知キー・未知フィールドは副作用前に 422（fail-closed）。
  使用モデルは既存 N6（config.frontend.generator.model）に記録。specs/19 §4.5 追補済み。
- **ビルダー UI**: プラン確認（生成開始）画面にモデル選択。既定 gpt-oss-120b・速度/品質の
  目安ラベル（i18n ja/en）・localStorage 復帰（未知値は既定へフォールバック）。
- **環境依存値**: GEN_SHARED_PROFILE / GEN_SHARED_COMPARTMENT_OCID（.env のみ・コミット禁止。
  未設定 = 共有テナンシモデルは fail-closed で使用不可）。

## 5.5 codex-review 指摘への対応と ZDR の実測（review-2 → 修正 → 再検証）

review-2 の blocker「responses のサーバ側保存（store）を境界で禁止できていない」への対応で
判明した事実と最終設計:

- **本上流（ORASEJAPAN 共有テナンシ）は Zero Data Retention**: store 未指定で作成した
  response の echo が `"store": false`（プローブ実測）。サーバ側に永続項目は作られない。
- プロキシは fail-closed を明示化: 明示 `store:true` / `conversation` /
  `previous_response_id` は **403 persistence_not_allowed**、canonical では常に
  `store:false` を強制（上流既定に依存しない）。
- **副次発見（先在の非決定バグ）**: @ai-sdk/openai はフローによって過去 turn の reasoning を
  `{"type":"item_reference","id":"rs_..."}` で echo する。ZDR 上流はこれを解決できず
  「Items are not persisted for Zero Data Retention organizations」の 404 → 生成が
  **非決定に失敗**（同条件 4 回中 2 回失敗を実測）。`include: reasoning.encrypted_content`
  の往復も本上流では成立しない（実測）。→ プロキシで `item_reference` / bare `reasoning`
  item を input から除去して無状態化（function call/output・message は内容完結で残す）。
  修正後の再生成（表 #4）で決定的に PASS。
- provider `@ai-sdk/openai@4.0.9` に版固定（N6 の generator.provider に記録 — 表 #4 で実証）。
  生成イメージへの事前導入は residual（Container Instance 化タスクの範囲）。
- 上流未到達/タイムアウトは 502 upstream_unreachable / 504 upstream_timeout に写像（契約テスト）。
