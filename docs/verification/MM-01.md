## 実装

| 層 | 内容 |
|---|---|
| モデル | `ModelDef.vision` フラグ（gemini-2.5-pro/flash）+ `llama-3.2-90b-vision` 新規登録。`/api/chat/models` に `vision` |
| API | `ChatRequest.images`（data URI、≤10枚・各2MB・計10MB）。visionモデル必須/agent・rag併用不可/最終userのみ(422系)。最終userメッセージをcontent partsへ変換（chat completionsは素通しで届く）。**履歴永続化はテキストのみ**（画像は当該ターンのみ — v1制約） |
| チャットUI | visionモデル選択時のみ📎表示（≤4枚）。長辺1024px JPEG(q0.85)へ縮小。入力欄上にチップ（削除可）、ユーザーバブルにサムネイル |
| 映像分析 | `/video` ページ（ナビ「映像分析」）: 動画をブラウザ内で等間隔4〜10フレームへcanvas変換（長辺768px。**動画自体は送信しない**）→ `/api/chat/stream` のimagesでステートレス分析。専用API追加なし |

## E2E結果

1. **ローカルAPI**: 移動する赤い円の6フレーム → gemini-2.5-flashが「赤い円が画面の左側から右側へと移動しています」と時系列を正しく説明
2. **ブラウザ実機（Playwright）**:
   - 画像チャット: 📎→添付→送信 → ユーザーバブルにサムネイル、応答は図形2つ（赤い円・青い四角）と文字「OCI 26」を正答（`img/MM-01-chat-image.png`）
   - 映像分析: **canvas+MediaRecorderでテスト動画(webm 3秒)をブラウザ内生成して投入** → 6フレーム抽出→分析が動作（`img/MM-01-video-ui.png`）
3. **実環境（GW経由・M2Mトークン、イメージ0.22.0）**: 6フレーム分析が「赤い円が左から右へ移動しています」と正答
4. バリデーション単体テスト6件追加（vision必須/agent併用不可/データURI/サイズ/最終ロール/モデル一覧フラグ）→ API計79件pass

## 制約・既知事項

- 画像はターン限り（履歴に画像は残らない。マルチターンで画像を参照し続ける場合は再添付）
- gpt-oss等の非visionモデルでは📎自体が出ない（誤送信は422でも防御）
- 動画はブラウザ処理のため大きなファイルでも帯域負荷なし。ただしコーデックはブラウザ依存（mp4/webm等）
