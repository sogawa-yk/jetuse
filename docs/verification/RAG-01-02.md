# RAG-01/02 検証レポート: ファイル管理 + RAGチャット（引用付き）

日付: 2026-06-11
仕様: specs/09-rag.md
状態: **実機E2E完了**（イメージ 0.8.1→0.9.0、migration 005適用、SPA同時デプロイ）。UIの操作感レビューはユーザー確認待ち

## 実装

- **RAG-01**: ユーザーごとのVector Storeを遅延作成（CP作成→completed待ち→DP伝播待ち、SPIKE-03準拠）。`RAG_STORES`/`RAG_FILES`（migration 005、状態の正はADB）。multipartアップロード（pdf/txt/md・20MB上限・docxは理由付きで明示拒否）→ Object Storage原本バックアップ（`rag/{owner}/...`）→ Files API → ファイル単位でVS登録。一覧時にprocessing行だけDPへ状態照会。削除はVS→Files→OS→ADBの順に同期
- **RAG-02**: `/api/chat/stream` に `rag: true`（Responses系のみ、未アップロード時は400）。SPIKE-03bのツール強制instructionsを自動付与し、`include=file_search_call.results`。SSEに `{"citations":[{file_id, filename, score}]}` イベントを追加（results+annotationsから重複排除・最大スコア採用）
- UI `/rag`: ファイル一覧（アップロード・状態バッジ・5秒ポーリング・削除）+ チャット（ストリーミング・停止・Md描画・**引用元チップ（ファイル名+スコア）**）。ホームにRAGカード追加
- 依存追加: python-multipart。CI環境変数に `RAG_BUCKET`（jetuse-dev-app-data流用）

## 追補（2026-06-11 間欠エラー対応、イメージ0.9.1）

ユーザー報告「⚠ Expecting property name enclosed in double quotes」= openai SDKが上流の非JSONペイロード（OCIの一時エラー形式）の解析に失敗した生エラー。stream_chatにJSONDecodeError個別処理を追加: 未出力なら1回自動リトライ / 出力途中なら「上流応答の解析に失敗しました（一時的なエラーの可能性）。再生成をお試しください」/ traceback記録（次回発生時の特定用）。12連続RAGクエリでは再現せず（間欠性）。pytest 36件パス。
