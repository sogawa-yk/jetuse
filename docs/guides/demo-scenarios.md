# プリセールス デモシナリオ集（DOC-04）

OCI版JetUseの強みを実機で見せるための台本。各シナリオは事前準備・所要時間・話す順・想定問答つき。
合計フルデモ約25分（個別に切り出し可）。**全シナリオの前提: dev ADBが起動済み**（夜間停止対策）。

---

## シナリオ1: 社内文書RAG（約5分）

**狙い**: 「自社文書に基づく根拠付き回答」をノーコードで。Object Storage原本＋Vector Storeの二重管理。

| 手順 | 操作 | 見せどころ |
|---|---|---|
| 準備 | 社内規程PDF（経費・就業規則等）を数点用意 | — |
| 1 | RAGチャットで文書アップロード→取り込み完了を待つ | アップロードのみで準備完了 |
| 2 | 「経費精算の上限は？」等を質問 | **引用チップ**（出典ファイル+スコア）付きで回答 |
| 3 | 文書に無いことを質問 | 「該当なし」と正直に答える（ハルシネーション抑制） |
| 4 | バックエンドをSelect AIに切替えて同じ質問 | DB内RAG（OCIだけの選択肢）と比較できる |

想定問答: 「精度は?」→ SPIKE-08で10/10正答・引用スコア実測（`docs/comparison/rag-backends.md`）。

---

## シナリオ2: 売上データのNL2SQL（約6分）

**狙い**: 自然言語→SQL→実行を、生成SQLの確認を挟んで安全に。差別化の山場。

| 手順 | 操作 | 見せどころ |
|---|---|---|
| 準備 | SHサンプルスキーマ（同梱） | — |
| 1 | DBチャットで「2001年の販売チャネル別売上を教えて」 | NL→**SQLを生成して提示**（実行前に確認できる） |
| 2 | 生成SQLを確認して実行 | 読取専用ユーザー・SELECT限定・200行/30秒ガード |
| 3 | 「グラフ化」 | LLMが適切なチャート種別を提案しChart.js描画 |
| 4 | SQL Search と Select AI を切替比較 | 正確性 vs 速度のトレードオフを実演 |

想定問答: 「危険なSQLは?」→ 多層ガード（読取専用ユーザー＋構文チェック＋タイムアウト）。
「精度?」→ SQL Search 10/10・Select AI 8/10（`docs/comparison/nl2sql-backends.md`）。

## シナリオ3: 議事録の自動生成（約5分）

**狙い**: 音声→話者分離付き文字起こし→議事録/FAQ/記事への整形を一気通貫。

| 手順 | 操作 | 見せどころ |
|---|---|---|
| 準備 | 会議音声（mp3/wav、複数話者） | — |
| 1 | 議事録ページで音声アップロード | バッチ文字起こし（話者分離）が走る |
| 2 | 完了後トランスクリプト表示 | **話者ごとに色分け**＋タイムスタンプ |
| 3 | テンプレート「議事録」で生成 | 決定事項・担当付きTODO・時系列を抽出（ストリーミング） |
| 4 | 「FAQ」「記事」に切替えて再生成 | 同じ素材から用途別に整形 |

想定問答: 「リアルタイムは?」→ リアルタイム文字起こし画面も実演可（partialなし=Whisper制約を説明）。
「日本語音声は?」→ Whisperで対応、TTSはPhoenix5ボイス。

## シナリオ4: エージェント＋MCP連携（約6分）

**狙い**: ツール・外部MCP・複数フレームワークを切替えられる拡張性。

| 手順 | 操作 | 見せどころ |
|---|---|---|
| 1 | エージェントビルダーで作成（実装=OpenAI Agents SDK） | **業界標準SDKがそのまま動く**（既定エンジン） |
| 2 | ツール（Web検索＋DB照会）を有効化、MCP（deepwiki等）も登録 | 社内データ＋外部知識の複合 |
| 3 | 「最新のOCI新機能を調べて、社内の利用状況と照らして」等 | 多段ツール実行（承認フロー or 自動） |
| 4 | 同じ構成を LangGraph / native に切替 | **エンジンを選べる**（顧客のスタックに合わせられる） |

想定問答: 「どのフレームワークを使うべき?」→ `docs/comparison/agent-frameworks.md` の顧客類型別推奨。
「承認は?」→ 人間承認フロー（HITL）をRunState往復で実現。
